# 项目二：供应链日报/周报 AI 辅助生成与自动推送工作流

## 项目概述

团队每日/每周需向管理层输出供应链运营报告，涵盖缺货率、履约时效、异常商家、改善进展等多项指标。
人工编写耗时 2~3 小时/次，且格式不统一、容易遗漏关键数据。

本工作流实现：**自动取数 → 指标计算 → AI 分析生成 → Jinja2 模板渲染 → 邮件/IM 自动推送**，
将报告制作从纯手工升级为 AI 辅助 + 人工审核模式。

## 工作流架构

```
  ┌────────────── Cron 定时调度 ──────────────┐
  │   日报: 每日 08:30    周报: 每周一 09:00    │
  └────────────────────┬──────────────────────┘
                       │
                       ▼
  ┌─────────────────────────────────────────────┐
  │              report_engine.py               │
  │                                             │
  │  ┌──────────┐   ┌──────────┐   ┌─────────┐ │
  │  │ 1. 取数   │──▶│ 2. 计算   │──▶│ 3. 渲染  │ │
  │  │ SQL查询   │   │ 10+ KPI  │   │ Jinja2  │ │
  │  │ FBI API  │   │ 环比同比  │   │ HTML模板 │ │
  │  └──────────┘   └──────────┘   └────┬────┘ │
  │                                     │      │
  │  ┌──────────┐   ┌──────────────────▼────┐ │
  │  │ 5. 推送   │◀──│ 4. AI 分析段落生成     │ │
  │  │ SMTP邮件  │   │ (Prompt → AI → 文本)   │ │
  │  │ 企微通知   │   └───────────────────────┘ │
  │  └──────────┘                              │
  └─────────────────────────────────────────────┘
```

## AI 工具参与环节

| 环节 | AI 工具 | 具体使用方式 |
|------|---------|------------|
| Python 脚本编写 | Codex | report_engine.py 全量代码辅助生成 |
| Jinja2 模板编写 | Codex | HTML 模板语法生成与调试 |
| 分析洞察生成 | Codex | 将结构化指标输入 Prompt，AI 自动生成分析段落 |
| 邮件模板 | Codex | HTML 邮件格式生成，兼容 Outlook/网页版 |

## 实体文件清单

| 文件 | 说明 |
|------|------|
| `scripts/report_engine.py` | 报告生成引擎（取数→计算→渲染→推送） |
| `templates/daily_report.html` | 日报 Jinja2 模板 |
| `templates/weekly_report.html` | 周报 Jinja2 模板 |
| `config/config.yaml` | 指标定义、推送配置 |
| `samples/daily_report_sample.html` | 日报产出样例 |
| `samples/weekly_report_sample.html` | 周报产出样例 |

## 项目效果

- 单次报告制作时间：**2~3 小时 → 10~15 分钟**（含人工审核），效率提升 **90%**
- 实现 7×24 全自动运行，节假日无需人工值班
- 报告格式统一，数据准确率 100%（自动取数无手工误差）
