# 项目三：商家活动库存编辑审批自动化工作流

## 项目概述

速卖通大促期间（双11、双12、618等），商家频繁需要修改活动报名库存。
原有流程依赖人工邮件/钉钉逐级审批：流程长（4~8小时）、易遗漏、无留痕、无统计。

本工作流基于 **宜搭低代码平台** 搭建标准化审批流，配合 Python API 回写脚本，
实现：**商家提交 → 表单自动校验 → 规则引擎分级 → 多级审批 → API 自动回写 → 通知推送** 全链路自动化。

## 工作流架构

```
  ┌────────────── 商家侧 ──────────────┐
  │                                    │
  │  商家登录 → 填写库存修改申请表单      │
  │  (宜搭表单: SKU、活动、库存量、原因)   │
  │                                    │
  └────────────────┬───────────────────┘
                   │ 表单自动校验
                   │ (库存>=0, 变更<=上限)
                   ▼
  ┌─────────────────────────────────────┐
  │         宜搭规则引擎 (分级路由)       │
  │                                     │
  │  变更 < 100件  ──→  主管审批 (L1)   │
  │  变更 100~500  ──→  运营经理 (L2)   │
  │  变更 > 500件  ──→  PM审批 (L3)    │
  │                                     │
  └────────────────┬────────────────────┘
                   │
          ┌────────┴────────┐
          ▼                 ▼
     ┌─────────┐      ┌─────────┐
     │ ✅ 通过  │      │ ❌ 驳回  │
     └────┬────┘      └────┬────┘
          │                │
          ▼                ▼
  ┌──────────────┐  ┌──────────────┐
  │ API 自动回写  │  │ 驳回通知 +    │
  │ 库存数据      │  │ 理由反馈     │
  └──────┬───────┘  └──────────────┘
         │
         ▼
  ┌──────────────────┐
  │ 钉钉/企微通知      │
  │ (申请人 + 运营群)  │
  └──────────────────┘
```

## AI 工具参与环节

| 环节 | AI 工具 | 具体使用方式 |
|------|---------|------------|
| 表单 JSON Schema | Codex | 辅助梳理业务字段与校验规则，快速生成宜搭兼容的 Schema |
| 审批流条件分支 | Codex | 编写宜搭 JS 扩展代码，实现分级路由逻辑 |
| API 回写脚本 | Codex | 辅助编写 inventory_callback.py 的 HTTP 请求、重试、日志 |
| 通知模板 | Codex | 生成不同审批结果的钉钉/企微消息模板 |

## 实体文件清单

| 文件 | 说明 |
|------|------|
| `yida/form_schema.json` | 宜搭库存编辑申请表单 Schema |
| `yida/approval_flow_config.json` | 审批流节点配置 |
| `scripts/inventory_callback.py` | 审批通过后 API 回写脚本 |
| `samples/notification_templates.md` | 四种场景的通知模板样例 |

## 项目效果

- 审批时效：人工 **4~8 小时 → 系统 < 1 小时**
- 100% 审批留痕，合规可追溯
- 商家满意度提升，大促备货响应速度加快
